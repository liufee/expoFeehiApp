# SQLite 管理器测试指南

## 快速测试步骤

### 1. 准备测试数据库

你可以使用以下方法创建一个测试用的 SQLite 数据库：

#### 方法一：使用命令行创建
```bash
# 在终端中执行
sqlite3 test.db <<EOF
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    email TEXT,
    age INTEGER,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

INSERT INTO users (name, email, age) VALUES 
('张三', 'zhangsan@example.com', 25),
('李四', 'lisi@example.com', 30),
('王五', 'wangwu@example.com', 28),
('赵六', 'zhaoliu@example.com', 35),
('孙七', 'sunqi@example.com', 22);

CREATE TABLE products (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    product_name TEXT NOT NULL,
    price REAL,
    stock INTEGER DEFAULT 0
);

INSERT INTO products (product_name, price, stock) VALUES
('iPhone', 6999.00, 100),
('MacBook', 12999.00, 50),
('iPad', 4999.00, 75);
EOF
```

#### 方法二：使用应用现有数据库
- 找到应用中使用的数据库文件（通常在 `Documents` 目录下）
- 通过文件管理器导出数据库文件
- 用于测试

### 2. 在应用中测试

#### 基础功能测试

1. **打开 SQLite 管理器**
   - 启动应用
   - 进入"工具"页面
   - 点击底部导航栏的"SQLite管理"标签

2. **选择数据库**
   - 点击"选择数据库文件"按钮
   - 选择准备好的测试数据库文件
   - 确认显示"已加载数据库: xxx.db"提示

3. **查看表列表**
   - 左侧应该显示所有表（users, products 等）
   - 每个表旁边显示行数
   - 确认表数量正确

4. **浏览表数据**
   - 点击 "users" 表
   - 右侧显示列头：id, name, email, age, created_at
   - 显示数据行，应该有 5 条记录
   - 主键字段（id）应该有钥匙图标

5. **测试分页**
   - 如果数据超过 20 条，测试分页功能
   - 点击下一页按钮
   - 点击上一页按钮
   - 确认页码显示正确

6. **切换表**
   - 点击 "products" 表
   - 确认数据显示正确
   - 返回点击 "users" 表
   - 确认能正常切换

#### 自定义查询测试

1. **打开查询窗口**
   - 点击右上角的代码图标（</>）
   - 确认弹出查询输入框

2. **执行简单查询**
   ```sql
   SELECT * FROM users
   ```
   - 点击"执行查询"
   - 确认显示所有用户数据

3. **执行带条件查询**
   ```sql
   SELECT * FROM users WHERE age > 25
   ```
   - 确认只显示年龄大于 25 的用户

4. **执行字段选择查询**
   ```sql
   SELECT name, email FROM users
   ```
   - 确认只显示 name 和 email 两列

5. **执行排序查询**
   ```sql
   SELECT * FROM users ORDER BY age DESC
   ```
   - 确认数据按年龄降序排列

6. **返回表浏览**
   - 点击"返回"按钮
   - 确认回到之前的表浏览状态

7. **测试安全限制**
   - 尝试执行 UPDATE 或 DELETE 语句
   - 确认系统提示"只允许执行SELECT查询"

#### 边界情况测试

1. **空表测试**
   - 创建一个空表
   - 确认显示"该表没有数据"

2. **大数据量测试**
   - 创建一个包含 100+ 条记录的表
   - 测试分页是否正常
   - 检查滚动性能

3. **特殊字符测试**
   - 插入包含特殊字符的数据（中文、emoji、引号等）
   - 确认显示正常

4. **NULL 值测试**
   - 确保 NULL 值显示为 "NULL"
   - 不会导致崩溃

5. **长文本测试**
   - 插入很长的文本字段
   - 确认会自动截断显示
   - 不会破坏布局

6. **关闭数据库**
   - 点击右上角关闭按钮
   - 确认界面回到初始状态
   - 可以重新选择数据库

### 3. 跨平台测试

#### iOS 测试
- 在 iOS 模拟器上测试
- 在真机上测试
- 确认文件选择器正常工作
- 检查界面适配

#### Android 测试
- 在 Android 模拟器上测试
- 在真机上测试
- 确认文件选择器正常工作
- 检查权限请求

### 4. 性能测试

1. **大数据库加载**
   - 测试 50MB+ 的数据库文件
   - 记录加载时间
   - 检查内存使用情况

2. **多表切换**
   - 快速切换不同的表
   - 确认响应速度
   - 检查是否有内存泄漏

3. **复杂查询**
   - 执行 JOIN 查询
   - 执行子查询
   - 检查执行时间

### 5. 错误处理测试

1. **无效文件测试**
   - 尝试选择非 SQLite 文件
   - 确认显示友好错误提示
   - 应用不崩溃

2. **损坏数据库测试**
   - 尝试打开损坏的数据库文件
   - 确认错误处理得当

3. **SQL 语法错误**
   - 输入错误的 SQL 语句
   - 确认显示具体错误信息
   - 应用不崩溃

### 6. UI/UX 测试

1. **深色模式**
   - 切换到深色模式
   - 确认所有元素颜色正确
   - 文字清晰可读

2. **横屏模式**
   - 旋转设备到横屏
   - 确认布局自适应
   - 所有功能正常

3. **不同屏幕尺寸**
   - 在小屏幕设备上测试
   - 在大屏幕设备上测试
   - 确认布局合理

## 已知问题和限制

1. 当前版本不支持数据编辑
2. 超大数据库（500MB+）可能加载缓慢
3. 复杂查询可能需要优化
4. 不支持事务操作
5. 不支持数据库结构修改
