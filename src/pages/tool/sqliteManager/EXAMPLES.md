## 常用查询示例

### 1. 查看所有数据
```sql
SELECT * FROM users
```

### 2. 条件筛选
```sql
SELECT * FROM users WHERE age > 25
```

### 3. 选择特定字段
```sql
SELECT name, email FROM users
```

### 4. 排序
```sql
SELECT * FROM users ORDER BY age DESC
```

### 5. 限制数量
```sql
SELECT * FROM users LIMIT 10
```

### 6. 模糊搜索
```sql
SELECT * FROM users WHERE name LIKE '%张%'
```

### 7. 多表 JOIN
```sql
SELECT u.name, p.product_name 
FROM users u 
JOIN orders o ON u.id = o.user_id
JOIN products p ON o.product_id = p.id
```

### 8. 聚合统计
```sql
SELECT COUNT(*) as total, AVG(age) as avg_age 
FROM users
```

## 最佳实践

1. **选择合适的数据库文件**
   - 确保文件未损坏
   - 文件大小适中（< 100MB 为佳）

2. **使用有效的 SQL 查询**
   - 先在小数据集上测试
   - 避免过于复杂的查询

3. **合理使用分页**
   - 大数据表务必使用分页
   - 避免一次性加载过多数据

4. **及时关闭数据库**
   - 使用完毕后点击关闭按钮
   - 释放内存资源

5. **备份重要数据**
   - 虽然只读操作很安全
   - 但仍建议提前备份

## 故障排除

### 问题: 无法选择数据库文件
**解决**: 
- 确认文件格式正确 (.db, .sqlite, .sqlite3)
- 检查文件权限
- 尝试重新导出数据库

### 问题: 数据显示乱码
**解决**:
- 确认数据库编码为 UTF-8
- 检查特殊字符是否正确存储

### 问题: 查询执行缓慢
**解决**:
- 优化 SQL 查询
- 添加适当的索引
- 减少返回的数据量

### 问题: 应用崩溃
**解决**:
- 检查数据库文件是否损坏
- 避免超大文件（> 500MB）
- 重启应用重试

## 提示和技巧

💡 **提示 1**: 使用 PRAGMA 命令查看表结构
```sql
PRAGMA table_info(users)
```

💡 **提示 2**: 查看所有表名
```sql
SELECT name FROM sqlite_master WHERE type='table'
```

💡 **提示 3**: 查看表的创建语句
```sql
SELECT sql FROM sqlite_master WHERE name='users'
```

💡 **提示 4**: 快速定位数据
- 使用 WHERE 条件精确筛选
- 避免全表扫描

💡 **提示 5**: 导出数据
- 当前版本暂不支持导出
- 可以使用桌面工具如 DB Browser for SQLite
