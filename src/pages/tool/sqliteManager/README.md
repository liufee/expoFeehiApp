# SQLite 管理器

这是一个用于在移动设备上调试和管理 SQLite 数据库的工具。

## 功能特性

### 1. 数据库文件选择
- 支持从设备中选择 `.db`、`.sqlite`、`.sqlite3` 等格式的数据库文件
- 自动解析并显示数据库中的所有表

### 2. 表浏览
- 左侧显示所有表的列表
- 每个表显示名称和行数
- 点击表名即可查看数据

### 3. 数据展示
- 自动显示字段名作为列头
- 主键字段会有特殊标记（钥匙图标）
- 数据以表格形式展示，支持交替行颜色
- 长文本会自动截断显示

### 4. 分页功能
- 默认每页显示 20 条记录
- 支持上一页/下一页导航
- 显示当前页码和总页数
- 显示总行数

### 5. 自定义 SQL 查询
- 点击右上角的代码图标可以打开查询窗口
- 支持执行 SELECT 查询语句
- 为了安全起见，只允许执行查询操作（不允许修改、删除等）
- 查询结果会以表格形式展示
- 可以返回到表浏览模式

## 使用方法

### 基本使用流程

1. **打开 SQLite 管理器**
   - 在工具页面底部导航栏点击 "SQLite管理" 标签

2. **选择数据库文件**
   - 点击 "选择数据库文件" 按钮
   - 从设备中选择要查看的 SQLite 数据库文件
   - 系统会自动加载数据库中的所有表

3. **浏览表数据**
   - 在左侧表列表中点击任意表名
   - 右侧会显示该表的数据
   - 使用底部的分页控件切换页面

4. **执行自定义查询**
   - 点击右上角的代码图标（</>）
   - 输入 SQL SELECT 查询语句
   - 例如：`SELECT * FROM users WHERE age > 18`
   - 点击 "执行查询" 按钮
   - 查看查询结果
   - 点击 "返回" 按钮回到表浏览模式

5. **关闭数据库**
   - 点击右上角的关闭图标（×）
   - 可以重新选择其他数据库文件

## 技术实现

### 使用的库
- `expo-sqlite`: 用于操作 SQLite 数据库
- `expo-document-picker`: 用于选择数据库文件
- `@expo/vector-icons`: 提供图标支持

### 安全措施
- 只允许执行 SELECT 查询，防止误操作修改数据
- 使用参数化查询防止 SQL 注入
- 数据库连接在使用完毕后正确关闭

### 性能优化
- 分页加载数据，避免一次性加载大量数据
- 每页默认显示 20 条记录，可调整
- 懒加载表数据，只在需要时查询

## 注意事项

1. **文件大小限制**
   - 过大的数据库文件可能导致加载缓慢
   - 建议用于调试中小型数据库

2. **内存管理**
   - 使用完毕后记得关闭数据库连接
   - 避免同时打开多个数据库文件

3. **兼容性**
   - 支持 iOS 和 Android 平台
   - 需要在真机或模拟器上测试

4. **权限要求**
   - 需要文件访问权限来选择数据库文件
   - Expo Go 环境下可能需要额外配置

## 常见问题

### Q: 为什么无法选择数据库文件？
A: 确保文件是有效的 SQLite 数据库格式，并且具有正确的扩展名（.db, .sqlite, .sqlite3 等）

### Q: 数据显示不完整怎么办？
A: 单元格内容过长时会自动截断，这是为了避免界面混乱。可以使用自定义查询来查看特定字段

### Q: 能否编辑数据？
A: 当前版本仅支持查看，不支持编辑。这是为了防止误操作导致数据损坏

### Q: 如何查看表结构？
A: 列头会显示所有字段名，主键字段会有钥匙图标标记
