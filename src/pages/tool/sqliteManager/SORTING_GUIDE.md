# 排序功能快速使用指南

## 🎯 功能概述

现在你可以点击任意列头对数据进行排序，并通过图标清楚地看到当前的排序状态。

## 📋 使用步骤

### 1️⃣ 基本排序

```
操作步骤：
1. 打开 SQLite 管理器
2. 选择一个数据库文件
3. 点击左侧的表名
4. 点击任意列头进行排序
```

**示例：**
```
原始数据：                    点击 "age" 列头后（升序）：
┌────┬──────┬─────┐          ┌────┬──────┬─────┐
│ id │ name │ age │          │ id │ name │ age │
├────┼──────┼─────┤          ├────┼──────┼─────┤
│ 1  │ 张三 │ 30  │    →     │ 3  │ 王五 │ 22  │
│ 2  │ 李四 │ 25  │          │ 2  │ 李四 │ 25  │
│ 3  │ 王五 │ 22  │          │ 1  │ 张三 │ 30  │
└────┴──────┴─────┘          └────┴──────┴─────┘
                                    ⬆️
                              age 列高亮 + 向上箭头
```

### 2️⃣ 切换排序方向

```
操作步骤：
1. 已经按某列排序（如 age 升序）
2. 再次点击同一列头
3. 排序方向切换为降序
```

**示例：**
```
升序排序：                    再次点击 "age" 后（降序）：
┌────┬──────┬─────┐          ┌────┬──────┬─────┐
│ id │ name │ age │          │ id │ name │ age │
├────┼──────┼─────┤          ├────┼──────┼─────┤
│ 3  │ 王五 │ 22  │    →     │ 1  │ 张三 │ 30  │
│ 2  │ 李四 │ 25  │          │ 2  │ 李四 │ 25  │
│ 1  │ 张三 │ 30  │          │ 3  │ 王五 │ 22  │
└────┴──────┴─────┘          └────┴──────┴─────┘
      ⬆️                            ⬇️
   向上箭头                      向下箭头
```

### 3️⃣ 切换排序列

```
操作步骤：
1. 已经按某列排序（如 age）
2. 点击另一列头（如 name）
3. 清除之前的排序，按新列排序
```

**示例：**
```
按 age 排序：                 点击 "name" 后：
┌────┬──────┬─────┐          ┌────┬──────┬─────┐
│ id │ name │ age │          │ id │ name │ age │
├────┼──────┼─────┤          ├────┼──────┼─────┤
│ 3  │ 王五 │ 22  │    →     │ 2  │ 李四 │ 25  │
│ 2  │ 李四 │ 25  │          │ 3  │ 王五 │ 22  │
│ 1  │ 张三 │ 30  │          │ 1  │ 张三 │ 30  │
└────┴──────┴─────┘          └────┴──────┴─────┘
      ⬆️                            ⬆️
   age 列高亮                  name 列高亮
```

## 🔍 视觉提示

### 列头状态说明

| 状态 | 外观 | 说明 |
|------|------|------|
| 未排序 | 普通颜色文本 | 该列当前不是排序依据 |
| 升序排序 | **主题色文本** + ⬆️ | 按该列升序排列 |
| 降序排序 | **主题色文本** + ⬇️ | 按该列降序排列 |
| 主键列 | 🔑 钥匙图标 | 该列是表的主键 |

### 图标组合示例

```
普通列（未排序）：
  name

主键列（未排序）：
  id 🔑

普通列（升序）：
  age ⬆️

普通列（降序）：
  age ⬇️

主键列（升序）：
  id 🔑 ⬆️

主键列（降序）：
  id 🔑 ⬇️
```


```
场景：大数据表需要排序浏览
操作：
1. 点击列头排序
2. 使用分页按钮翻页
3. 每页都保持相同的排序
```

### 技巧 4：重置排序
```
场景：想恢复原始顺序

结果：排序状态被重置
```

## ⚠️ 注意事项

### 1. NULL 值处理
```
升序排序时：
- NULL 值通常排在最前面
- 这是 SQLite 的默认行为

示例：
┌──────┬─────┐
│ name │ age │
├──────┼─────┤
│ 张三 │ NULL│  ← NULL 在最前
│ 李四 │ 25  │
│ 王五 │ 30  │
└──────┴─────┘
```

### 2. 数据类型影响
```
文本类型：按字典序排序
  "10" < "2"  （因为 "1" < "2"）

数字类型：按数值大小排序
  2 < 10

建议：确保数据类型正确
```

### 3. 中文排序
```
中文按 Unicode 编码排序
可能不是按拼音顺序

如需拼音排序，建议在查询时使用：
SELECT * FROM users ORDER BY name COLLATE LOCALIZED
```

## 🔧 常见问题

### Q1: 为什么排序后数据看起来没变化？
**A**: 可能原因：
- 数据本身已经是这个顺序
- 所有值都相同
- 只有很少的数据

### Q2: 如何恢复到原始顺序？
**A**: 
- 方法1：切换表再切回来
- 方法2：关闭数据库重新打开
- 方法3：记住原始的主键顺序，按主键排序

### Q3: 排序会影响性能吗？
**A**: 
- 小数据表（< 1000 行）：几乎无影响
- 中等数据表（1000-10000 行）：轻微影响
- 大数据表（> 10000 行）：建议使用索引

### Q4: 可以对多列同时排序吗？
**A**: 
- 当前版本不支持
- 可以使用自定义查询实现：
  ```sql
  SELECT * FROM users ORDER BY age DESC, name ASC
  ```

### Q5: 排序后导出数据会保持顺序吗？
**A**: 
- 当前版本不支持导出
- 未来版本会考虑此功能

## 🚀 高级用法

### 结合自定义查询
```sql
-- 在查询窗口中使用排序
SELECT name, age FROM users WHERE age > 25 ORDER BY age DESC

-- 多列排序
SELECT * FROM users ORDER BY department ASC, salary DESC

-- 表达式排序
SELECT *, (salary - bonus) as net_salary 
FROM employees 
ORDER BY net_salary DESC
```

### 利用索引优化
```sql
-- 如果经常按某列排序，可以创建索引
CREATE INDEX idx_users_age ON users(age);
CREATE INDEX idx_users_name ON users(name);

-- 复合索引用于多列排序
CREATE INDEX idx_users_dept_salary ON users(department, salary);
```
